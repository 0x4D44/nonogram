# JRN — Transcribed-solver probe verification: PUMPKIN + MILLENNI

**Date:** 2026-05-29
**Branch:** `main`
**Related:** `wrk_docs/2026.05.29 - HLD - NONOGRAM 16-bit to 32-bit mdtpw port.md`

## Method

Probe `_diag/solve_file.pas` = byte-identical transcription of the app's full
solver (QuickData / CheckEdge / TestRowFull / FollowRow / SingleTst /
Combination fixpoint) + the app's EXACT `FileLoadImplement` parser
(`NONOGRAM.PAS:1800-1951`) reading the `.NGM` at runtime. Consts bumped to
`MaxXAcross = MaxYAcross = 100`, `MaxNoElements = 40`. Compiled with prebuilt
`target/debug/mdtpw.exe` + i686 `mdtpw_rt.lib`.

Loader proven independently with `_diag/load_only.pas` (parses identically to
`solve_ref2.py`). External clue-check: `_diag/check_probe.py`.

## PUMPKIN (30×30) — CORRECT

- `solve_file.exe` (NGMFILE=`_puzzles\PUMPKIN.NGM`): passes=6, filled=485,
  empty=415, unknown=0. Probe `CheckViol` VIOLATIONS=0.
- External check (`check_probe.py` vs PUMPKIN.NGM): filled=485 unknown=0
  clue_violations=0 → CORRECT. Matches col-sum = row-sum = 485.
- Runtime: <1 s. ⇒ App solver solves PUMPKIN. Counts toward solvable.

## MILLENNI (75×55) — does NOT solve (unsatisfiable puzzle + solver hang)

- Loader OK (`load_only.exe`): XAcross=75 YAcross=55, cols 65-71 + rows 1-7
  non-empty, col-sum = row-sum = 256 — identical to `solve_ref2.py` parse.
- Full solve `solve_mill.exe`: loaded ("Loaded … XAcross=75 YAcross=55") then
  stuck in the solver >8.5 min, never printed a `passes=` summary; killed.
- MILLENNI.NGM is PROVABLY UNSATISFIABLE: every row except 1-7 and every column
  except 65-71 has an empty clue → forced all-empty → filled cells confined to
  rows 1-7 × cols 65-71 = 7×7 = 49-cell box, but clues demand 256 filled. Rows
  1-6 each demand 44-49 filled in only 7 available columns. No solution exists.
- The hang is the app's Combination strategy design limit (exhaustive placement
  enumeration ~C(32,16) on the 16-run row-6 / col-70 line in 75 cells), NOT a
  codegen issue (loader + 49-run/16-run arithmetic compile & run fine; PUMPKIN
  with a 17-run/15-element line solves correctly).

## Conclusion

PUMPKIN solvable. MILLENNI is a corrupt/contradictory file with no solution;
the faithful app-solver cannot solve it in practical time. No `NONOGRAM.PAS`
or `mdtpw/src` changes were made for this verification.
